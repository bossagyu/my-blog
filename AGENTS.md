# Agent Instructions（リポジトリ共通）

このファイルはリポジトリ全体に適用されます。下位ディレクトリに別の`AGENTS.md`がある場合はそちらを優先してください。
会話は日本語で行います。

## 作業優先順位
1. システム／デベロッパープロンプト > 本ファイル > その他のコンテキストの順に従うこと。
2. masterブランチに対して直接コミットしないこと。必ず新しいブランチを作成してから作業すること。
3. 変更を行った場合は必ず現在のブランチでコミットし、その後`make_pr`ツールを呼び出してPR文面を作成すること。
4. 追加の`AGENTS.md`がないか、作業パス配下を確認すること。

## ブログ出力方針（`content/blog`配下の既存記事に基づく）
- Hugo用Markdown記事を日本語で執筆する。語調は「です・ます調」で、手順や背景が簡潔に伝わる説明を心がける。
- フロントマターはTOML形式（`+++`で囲む）を使用し、以下を基本に含める。  
  - `title`: 日本語タイトル  
  - `date`: `YYYY-MM-DDTHH:MM:SS+09:00`形式のJST日時  
  - `draft`: falseをデフォルトで作成
  - `categories`: 配列形式でカテゴリを指定、指定できるカテゴリは `Engineering`、`Management`、`Blog` のいずれか
  - `tags`: 内容に即したタグ配列（ツール名やトピックを明示）を英語で作成、他の記事のタグを参考にする
- 本文構成は既存記事に合わせ、`## 概要`で導入を置き、その後は見出し（`##`/`###`）で章立てする。  
  コマンドやコードはフェンス付きコードブロック（言語指定あり）を使い、箇条書き・表・ショートコード（例: YouTube埋め込み）などMarkdown要素を適宜活用する。
- 内容は既存のブログ同様、ツールのセットアップ手順、実行例、考え方の整理、数式やサンプル結果など、エンジニアリング寄りの実用記事もしくは、マネジメントやスクラムに関する記述を含める。
- 日本語の記事を作成した場合、対応する英語記事も同時に作成する。英語記事のフロントマターは日本語記事を参考にし、`title`は英訳、`date`は日本語記事と同じ日時、`draft`はfalse、`categories`と`tags`も日本語記事に合わせる。本文は英語で執筆し、日本語記事の内容を忠実に反映させる。

## ツール利用
- 検索は`rg`を優先し、大規模ディレクトリでは`ls -R`や`grep -R`を避ける。
- この環境は非対話的なため、確認を求めず必要なコマンドを実行する。

## ビジュアル変更
- 目に見えるフロントエンド変更を行った場合、ブラウザツールが利用可能ならスクリーンショットを取得し、結果に添付する。
