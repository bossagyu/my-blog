**戦略**: 量産+改善サイクル（検索ボリュームを狙って週1本投稿、データで改善）

## 現状

- 記事数: 49記事（2023年12月〜2026年2月）
- カテゴリ: Engineering（メイン）、Management、Blog
- 月間PV: 数百程度、一部記事が検索にヒット

## 全体サイクル

```
① キーワード調査 → ② 記事作成（週1本） → ③ 公開・インデックス登録
     ↑                                            ↓
⑤ リライト・改善  ←  ④ Search Consoleでデータ分析（月1回）
```

月4本のうち、3本は新規記事、1本は既存記事のリライト。

## キーワード選定の基準

1. **検索意図が明確** — 「〇〇 方法」「〇〇 使い方」「〇〇 設定」など、具体的な行動意図がある
2. **競合が弱い** — Qiita/Zennの記事が古い、または情報が不十分なもの
3. **自分が書ける** — 実際に手を動かして検証できるテーマ

## タイトル設計のルール

- 「何を + どうする」を明確にする（例: 「RedisをDockerで起動する方法」）
- 検索キーワードをタイトルの前半に入れる
- 30〜40文字程度に収める

## 記事構成の標準テンプレート

```markdown
## 概要（何ができるか、誰向けか）
## 前提条件（環境・バージョン）
## 手順（ステップバイステップ）
## 動作確認
## まとめ（次のステップへの誘導）
```

## 記事の型

| 型 | 例 | 特徴 |
|---|---|---|
| セットアップ系 | 「〇〇のインストール・設定方法」 | 既存の得意パターン |
| 使い方系 | 「〇〇の使い方まとめ」 | 網羅的な解説 |
| 比較系 | 「〇〇 vs △△ 違いと選び方」 | 検索意図が明確 |
| トラブルシュート系 | 「〇〇 エラーの解決方法」 | ニッチで競合が少ない |

## 内部リンク戦略

- 関連する既存記事へのリンクを「まとめ」セクションに入れる
- 「前提条件」で環境構築記事へリンクする（例: Python記事 → uv記事へ誘導）

## 既存記事の改善プロセス

### Search Console分析（月1回）

| 指標 | 見るポイント | アクション |
|---|---|---|
| 表示回数が多い + CTRが低い | タイトル・descriptionが検索意図とズレている | タイトルとdescriptionをリライト |
| 掲載順位10〜30位 | もう少しで1ページ目に入れる | 記事内容の加筆・最新化 |
| クリック数が安定 | すでに強い記事 | 内部リンクのハブにする |

### リライトの優先順位

1. 掲載順位10〜30位の記事（最も投資対効果が高い）
2. 表示回数は多いがCTRが低い記事
3. 情報が古くなっている記事（バージョン変更等）

### リライト時のチェックリスト

- [ ] 最新バージョンでの動作確認
- [ ] descriptionの最適化（120〜160文字、キーワード含む）
- [ ] 関連する新記事への内部リンク追加
- [ ] `lastmod`フロントマターの追加

## 実行フェーズ

### Phase 1: 基盤整備（初週）

- 既存49記事間の内部リンクを整理・追加する

### Phase 3: データ駆動の改善（1ヶ月後〜）

- Google Search Consoleで検索パフォーマンスを確認
- 月1回、上位候補記事をリライト
